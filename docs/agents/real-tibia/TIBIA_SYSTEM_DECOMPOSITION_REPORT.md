### Existing-record conclusions

- No current record is deprecated or merged away in TSD-001.
- `combat`, `cyclopedia`, `market`, `prey`, `protocol`, `quests` and `spawns` remain broad compatibility/discovery umbrellas.
- `charms` and `spells` already behave as narrower durable modules; their parent relation is documented, not encoded through `depends_on`.
- `player-persistence` remains intact until TSD-002 separates connection, migration, transaction and restart/reload proof boundaries.

## Hierarchy decision

The current schema has no hierarchy field. TSD-001 does not add one because current `lookup-path`, `affected`, generated indexes and Upstream Intelligence already support deterministic many-to-many path mapping without parent metadata.

Hierarchy is therefore expressed by:

- stable umbrella IDs;
- child scope/description boundaries;
- this classification report and the decomposition program;
- overlapping verified path hints where appropriate;
- no fake `depends_on` edges.
